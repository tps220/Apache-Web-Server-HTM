529 /* BIG FAT WARNING: SSLModConfigRec has unusual memory lifetime: it is
530  * allocated out of the "process" pool and only a single such
531  * structure is created and used for the lifetime of the process.
532  * (The process pool is s->process->pool and is stored in the .pPool
533  * field.) Most members of this structure are likewise allocated out
534  * of the process pool, but notably sesscache and sesscache_context
535  * are not.
536  *
537  * The structure is treated as mostly immutable after a single config
538  * parse has completed; the post_config hook (ssl_init_Module) flips
539  * the bFixed flag to true and subsequent invocations of the config
540  * callbacks hence do nothing.
541  *
542  * This odd lifetime strategy is used so that encrypted private keys
543  * can be decrypted once at startup and continue to be used across
544  * subsequent server reloads where the interactive password prompt is
545  * not possible.
546 
547  * It is really an ABI nightmare waiting to happen since DSOs are
548  * reloaded across restarts, and nothing prevents the struct type
549  * changing across such reloads, yet the cached structure will be
550  * assumed to match regardless.
551  *
552  * This should really be fixed using a smaller structure which only
553  * stores that which is absolutely necessary (the private keys, maybe
554  * the random seed), and have that structure be strictly ABI-versioned
555  * for safety.
556  */
